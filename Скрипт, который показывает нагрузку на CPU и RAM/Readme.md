Что это такое?  
Небольшой монитор процессов, написанный на PowerShell, который отображает  
процессы с наибольшим потреблением CPU и RAM в режиме реального времени.  

Проект содержит две версии:  
* полноценную с расширенным выводом и статистикой,  
* минималистичную, для слабых машин и быстрого мониторинга.  

Возможности:  
* мониторинг процессов в реальном времени,  
* топ 10 процессов по ЦПУ,  
* топ 10 процессов по ОЗУ,  
* отображение PID,  
* отображение количества потоков,  
* цветовая подсветка (подсвечивает первые 3 наиболее нагруженных процесса),  
* настраиваемый интервал обновления (по умолчанию выставлено 10 сек.),  
* отображение общей статистики по нагрузке системы (в полной версии).  

Как работает?  
1. Получает список всех процессов через Get-Process.  
2. Вычисляет дополнительные параметры:  
* время процесса cpu,  
* использование озу,  
* количество потоков.  
3. Сортирует процессы по:  
* CPU  
* RAM  
4. Выводит топ n-ных процессов.  
5. Ждёт заданный интервал времени (10 сек.).  
6. Повторяет цикл.  
7. Обновление происходит в бесконечном цикле до остановки пользователем.  

Интервал обновления (в секундах):  
$interval = 10  

Количество отображаемых процессов:  
$topCount = 10  

Системные требования:  
PowerShell 5.1+ с правами администратора.
