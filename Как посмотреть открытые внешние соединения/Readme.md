Что это такое?  
Скрипт на PowerShell, который позволяет быстро прочекать все  
Established TCP-соединения на Windows-машине, группируя их по локальному адресу и показывая полезную инфу о каждом из них.  
Короче говоря, это - инструмент для быстрого монитора TCP-соединений на Windows-машине.  

Основные функции скрипта:
* Получение всех TCP-соединений в состоянии Established.
* Определение процесса, который держит соединение.
* Подсказки на тему того, что на каком порту висит - SOCKS5, Shadowsocks, OpenVPN, WireGuard и т.д.
* Группировка по локальному адресу для более наглядного отображения.

* Скорость. Скрипт делает один вызов Get-Process и создаёт карту процессов $processMap.
* Для каждого соединения не вызывается Get-Process повторно, что существенно экономит время при тысячах соединений.
* Если процесс завершился к моменту обработки, имя всё равно подставляется через Get-Process -ErrorAction SilentlyContinue.

* Масштабируемость. Может обрабатывать десятки тысяч TCP-соединений без заметного торможения.
* Подходит для серверов с большим количеством клиентов, прокси, VPN или балансеров нагрузки.

* Инфа. Показывает локальные и удалённые адреса, порты, процесс и подсказку по типу сервиса.
* Обеспечивает полный обзор соединений на хосте в одном выводе.
* Табличная структура.

* Права администратора для запуска не нужны. Просто открываем PowerShell, кидаем туда скрипт и запускаем.  
Ну, или через PowerShell ISE.
