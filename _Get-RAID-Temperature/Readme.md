Что это такое?  

Утилита на PowerShell для мониторинга температуры RAID-контроллера  
и отображения её в системном трее Windows с цветовой индикацией.  
Писалась в частности под Lenovo ThinkSystem RAID 930‑8i 2gb.  

Версия 6.  

Скрипт делает следующее:  
- использует утилиту storcli64.exe для получения температуры RAID-контроллера.  
- storcli64.exe в комплекте нет. Её предварительно нужно скачать вот отсюда: https://docs.broadcom.com/docs/1232743397  
и поместить на диск C:\storcli64.exe. По-другому работать не будет!  
- После запуска создаёт иконку в системном трее с текущей температурой.  
- Цвет текста иконки меняется в зависимости от температуры:  
	* Белый - нормальная температура (до 70 градусов Цельсия).  
	* Оранж - высокая температура (70-80 градусов Цельсия).  
	* Красный - критическая (выше 80 градусов Цельсия).  
	* Серый - ошибка или отсутствие данных.  
- Обновление температуры происходит автоматически каждые 30 секунд.  
Интервал указан в миллисекундах (30000 миллисекунд). Его можно настраивать.  
- Есть возможность вручную обновить температуру или выйти через контекстное меню приложения (пкм по иконке в трее).  
- Зачем нужно автоматическое нажатие на Enter и переход на новую строку в самом конце скрипта? [System.Console]::WriteLine()  
Без него подвисало контекстное меню.  
- В версии 6 добавлена проверка на наличие storcli как таковой по пути C:\storcli64.exe  

Требования:  
- Windows 10/11  
- PowerShell 5.1+  
- Утилита storcli64.exe по пути C:\storcli64.exe  
- .NET Framework по сути любой начиная с 2.0+  
- ОЗУ ~50 мб ( в диспетчере задач будет отображаться просто как PowerShell, или обработчик команд Windows )  

Установка:  
1) Проще всего запустить обычный PowerShell от админа и кинуть туда скрипт. Всё заработает.  
2) Если открыть PowerShell без прав админа, работать не будет.  
3) Если запускать его через PowerShell ISE, будет ряд проблем.  
Во-первых, скрипт будет очень долго закрываться.  
Причина медленного закрытия - бесконечный цикл Application.Run() + таймер + Stop в ISE,  
который не может мгновенно завершить форму и события таймера.  
В обычном powershell-e или bat-нике такого тормоза нет - потому что поток управления нормально  
завершает WinForms. Короче говоря, кнопка Stop в ISE будет ломать такие скрипты.  
И это не чинится. Вообще.  
А во-вторых, нас будет ждать ещё и ошибка необрабатываемого исключения.  
Она появляется потому что PowerShell уже закрыт, а таймер продолжает работать.  
Здесь только один вариант - нажать "Выход".  

Как закинуть всё это в автозагрузку?  
1) Кидаем скрипт на диск C:\Get-RAID-Temperature_v6_utf-8_+enter.ps1  
Скрипт должен быть в кодировке ANSI (Windows-1251).  
Если конвертировать его в UTF-8, или в UTF-8 BOM, работать, скорее всего, не будет.  
2) Кидаем батник в автозагрузку: Пуск - Программы - Автозагрузка: Get-RAID-Temperature-START_v2.bat  
3) Дополнительно, можно также открыть его (батник) через Notepad++ и убедиться, что стоит кодировка UTF-8.
У батника должна быть кодировка UTF-8, у скрипта - ANSI! 
4) Всё, после перезагрузки, автоматически появится окно с вопросом:  
"Разрешить этому приложению вносить изменения на вашем устройстве?" Говорим: "Да".  
И у нас появится счётчик температуры в трее. Всё.
